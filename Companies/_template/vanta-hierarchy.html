Current State: Disconnected Orgs, Duplicated Controls
Each org is a separate Vanta account with its own controls mapped to its own frameworks. No cross-org visibility, no shared standards, no central governance. Central teams manually compare controls via spreadsheets.

The problem: Each org defines controls independently. The same control (e.g., MFA Required) exists in Org A for SOC 2 and Org B for ISO 27001, but there's no connection between them. Central teams can't see gaps, can't enforce standards, and discover inconsistencies from auditors rather than the platform.
Bet 1: Actionable Cross-Org Visibility + Governance Lite
Introduces the Organizations Hub layer and a new cross-org control object. CCAs create hub-level control definitions mapped to frameworks and tagged by org type. The system matches these against existing org controls and surfaces a gap analysis. No push-down or enforcement yet.
What's new in Bet 1:
  • Organizations Hub layer connecting separate orgs
  • Hub-level control definitions (new cross-org data object) mapped to frameworks and tagged required/recommended
  • Automated gap analysis: system matches hub controls to existing org controls
  • Conversational hub interface for cross-org queries
  • Role-aware views and prioritized attention signals

Bet 2: Centralized Governance
Hub controls now push down to connected orgs. OCMs receive inherited controls, acknowledge, implement, or request exceptions. CCAs monitor rollout and enforce compliance deadlines. Exception workflow ships alongside inheritance because governance without flexibility won't get adopted.
What's new in Bet 2:
  • Control inheritance: hub controls push down to in-scope orgs automatically
  • OCM task queue: inherited controls appear as actionable items with deadlines
  • Exception request and approval workflow with audit trail
  • Rollout monitoring: per-org status tracking from the hub
  • Scoped to top 3 frameworks initially

Bet 3: Evidence Sharing
Optimization layer on top of Bet 2's governance foundation. Orgs share evidence and test results when requirements overlap across inherited controls. Reduces duplicated compliance work and increases switching cost. AI suggests evidence mappings when controls overlap.
What's new in Bet 3:
  • Cross-org evidence linking for shared/inherited controls
  • AI-suggested evidence mappings when controls overlap
  • Evidence status visibility from the hub
  • Shared evidence tagged and auditable across orgs
